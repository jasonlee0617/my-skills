# Decision Audit 中文说明

Decision Audit 是一个用于审查决策、方案和进行中工作的 Codex Skill。它通过结构化问题暴露信息盲区、认知遗漏、失败风险、创新机会和沟通低效，避免在证据不足时过早下结论。

## 核心能力

| 审查维度 | 核心问题 | 输出重点 |
|---|---|---|
| 信息盲区 | 当前最没有把握的事情是什么？ | 6–7 项重要缺口、形成原因、补证方式 |
| 认知漏洞 | 最大遗漏是什么，还有什么尚未意识到？ | 隐含假设、遗漏角色、二阶影响和框架偏差 |
| 失败预演 | 如果方案三个月后失败，最可能因为什么？ | 失败原因、早期信号和缓解措施 |
| 创意拓展 | 只能增加一项行业领先能力时选什么？ | 一项高价值能力、替代比较、成本和风险 |
| 沟通优化 | 如何提高本次对话和决策效率？ | 更合理的问题顺序、证据和决策格式 |

## 工作流程

1. 明确待审查的决策、目标、周期和约束。
2. 先读取现有文件和证据，再判断哪些信息缺失。
3. 使用五个维度进行审查。
4. 按影响与紧迫性排序，不平均对待所有发现。
5. 给出最少但最有效的后续行动。

## 使用示例

```text
$decision-audit
请严格审查当前 YOLO-LLM 机械臂抓取方案，重点检查信息盲区和三个月内最可能出现的失败原因。
```

## 输出结构

1. 决策与现有证据。
2. 最高影响的信息盲区。
3. 最大的未意识到的遗漏。
4. 三个月失败预演。
5. 一项领先能力建议。
6. 沟通改进方式。
7. 按优先级排列的行动项。

## 使用边界

- 不为了凑足数量而制造无关问题。
- 明确区分事实、推断、未知项和建议。
- 对不确定结论明确标注证据不足。
- 创意建议必须同时说明成本、依赖和风险。
